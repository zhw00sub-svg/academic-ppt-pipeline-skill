# PPT Standards

## Core principle

The deck must look like an editable academic presentation, not a sparse tech keynote and not a hospital poster.

## Two quality levels

### 最低过线

- editable `.pptx`
- no major overlap
- readable fonts
- clear evidence structure
- limited image-page mismatch

### 目标交付

- most slides need only light user edits
- image and text feel designed together, not assembled separately
- dense pages remain elegant and readable
- literature screenshots, pathway visuals, and text blocks feel like one system
- no slide falls back to generic template or empty keynote spacing

Default to `目标交付`.

## Page structure rules

- `Cover`: large Chinese title on one side, approved hero image on the other side.
- `Interpretation page`: allow dense text blocks if the content needs them.
- `Comparison page`: use native tables and keep text compact but readable.
- `Data page`: numbers must stay editable; charts and cards should be native shapes.
- `Pathway page`: image may supply the visual path, but labels should be native text.
- `Summary page`: compress to 3-4 strong conclusions plus one visual anchor.

## Typography floor

- Main cover title: usually `30pt+`
- Section title bar: around `24-26pt`
- Card title: around `16.5-18pt`
- Body text: usually `13.5-18pt` depending on density
- Footer references: around `8.5-10pt`

Never pursue elegance by dropping below conference readability.

## Density rules

- Slides may be full if the reference template is full.
- Half-screen text blocks are acceptable and often desirable.
- Leave breathing room, but do not over-preserve whitespace.
- If the slide still feels like a draft, it is not at the target standard yet.
- If a slide is still too empty after the core content is placed, add one of:
  - literature cover strip
  - screenshot from paper
  - evidence card
  - short interpretation block

## What must stay native in PPT

- titles
- data values
- trial names
- references
- arrows and labels that the speaker may need to revise
- page-specific clinical conclusions

## QA pass before delivery

1. Check for text overlap with images or shapes.
2. Check for footer collisions.
3. Check that text-heavy slides still use readable font sizes.
4. Check that image pages do not look more polished than the academic content pages.
5. Check that the full deck still belongs to one visual family.
