# Guided Intake

## Goal

Use a short step-by-step conversation to collect only the choices that materially affect the deck.

If the user has already clearly specified the workflow and output standard, skip this intake and move directly into production.

## Intake rule

- ask one focused question at a time
- prefer option-style choices
- recommend one option when possible
- avoid broad “tell me anything else” prompts

## Recommended intake order

### Step 1: Input mode

Ask which starting state applies:
- full materials available
- partial materials available
- topic only

### Step 2: Output target

Ask what the user needs first:
- full editable PPT
- outline first, then PPT
- visual style first, then content production

### Step 3: Visual anchor

Ask whether there is:
- a template PPT
- an approved benchmark image
- both
- neither

### Step 4: Evidence strictness

Ask whether the deck should be:
- literature-strict throughout
- literature-strict on data pages only
- structure first, literature fill later

### Step 5: Image generation usage

Ask whether to:
- use image generation only on key pages
- avoid image generation and stay fully native
- generate prompts only for external execution

## Once enough is known

Stop asking and start producing.

The skill should then:
- summarize the chosen path briefly
- run the workflow automatically
- continue until PPT delivery
